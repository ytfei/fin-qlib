## 🚀 使用方法

### 快速开始

```bash
# 基础回测（使用默认参数）
python scripts/run_backtest.py

# 小规模测试（建议首次使用）
python scripts/run_backtest.py --topk 10 --start 2025-01-02 --end 2025-01-10

# 自定义参数
python scripts/run_backtest.py --topk 30 --n-drop 5 --start 2025-01-01 --end 2025-03-31
```

### 可用策略

| 策略类型 | 说明 |
|---------|------|
| `topk_dropout` | 持仓 topk 只，每次调仓卖出 n_drop 只表现最差的 |
| `topk` | 持仓 topk 只，不主动卖出 |
| `soft_topk` | 软约束的 Topk 策略 |

## ⚠️ 注意事项

### 当前使用简化回测方法

由于 qlib backtest API 的一些兼容性问题，当前使用简化回测方法：

**特点**：
- ✅ 能够完成回测
- ✅ 生成每日收益和持仓
- ✅ 计算完整风险指标（夏普比率、最大回撤、信息比率等）
- ✅ 生成可视化图表（累计收益曲线、回撤曲线）
- ✅ 保存完整分析结果到 YAML

**局限**：
- 使用信号值作为收益代理（非真实交易收益）
- 基准收益计算可能不准确（使用简化方法）
- 回测结果仅供参考

### 获取真实回测结果

如果需要真实的交易回测（考虑成本、滑点等），建议：

1. **直接使用 Qlib 原生 API**：
```python
from qlib.contrib.evaluate import backtest_daily
from qlib.contrib.strategy import TopkDropoutStrategy

# 参考文档：https://qlib.readthedocs.io/
```

2. **使用专业回测平台**：
- 聚宽、米筐等专业平台
- 支持更精确的成本和滑点模型

### 简化回测的收益解读

当前回测返回的"收益率"实际上是：
- 预测信号值的平均（TopK 股票的平均预测分数）
- **不是**真实的交易收益率
- 可以作为策略表现的相对比较指标
